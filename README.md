## 📋 О проекте

### Бизнес-задача
Компания-организатор готовится к проведению первой лицензионной конференции TED. Заказчику нужно опереться на опыт прошлых мероприятий и ответить на вопросы:

- Какие темы популярны у зрителей?
- Какая оптимальная длительность выступления?
- Кто из выступающих наиболее интересен аудитории?
- Как зрители реагируют на разные темы (смех, аплодисменты)?

### Решение
Разработан интерактивный дашборд, который позволяет исследовать данные о конференциях TED за всё время существования проекта. Дашборд даёт возможность фильтровать данные, находить тренды и получать конкретные ответы на бизнес-вопросы.

## 📊 Структура дашборда

### 1. Блок управления (фильтры)
- **Страна** проведения конференции
- **Название конференции**
- **Тег выступления** (`main_tag`)
- **Дата записи** (диапазон)

### 2. Ключевые метрики (KPI)
| Метрика | Формула |
|---------|---------|
| Количество конференций | `COUNTD([events.conf_id])` |
| Количество выступлений | `COUNTUNIQUE([talk_id])` |
| Количество тегов | `COUNTUNIQUE([main_tag])` |
| Уникальное количество выступающих | `COUNTUNIQUE([author_id])` |

### 3. Аналитические блоки

#### 📌 Информация о конференциях
Таблица с группировкой по странам и конференциям:
- Средняя продолжительность выступления (в минутах)
- Максимальное количество просмотров
- Суммарное количество аплодисментов
- Количество выступающих на конференции

#### 📌 Топ-10 самых смешных выступлений
Линейчатая диаграмма, показывающая выступления с максимальным количеством смеха в зале (`laughter_count`).  
*Ранжирование: `RANK(SUM([laughter_count]), "desc")` + фильтр ≤10*

#### 📌 Топ-20 самых популярных тегов
Линейчатая диаграмма с темами (`main_tag`), которые чаще всего выбирали выступающие.  
*Ранжирование: `RANK(COUNT([talk_id]), "desc")` + фильтр ≤20*

#### 📌 Информация о выступающих
Таблица с данными о спикерах:
- Имя выступающего
- Род деятельности
- Суммарное количество аплодисментов

#### 📌 Информация о выступлениях
Детальная таблица со всеми выступлениями:
- Кликабельная ссылка на видео (`URL([url],[title])`)
- Название выступления
- Спикер
- Дата записи
- Количество просмотров
- Язык выступления
- Теги и метрики реакции зала

## 🧮 Технические детали

### Формулы ключевых метрик

```sql
-- Количество конференций (без дублей при JOIN)
COUNTD([events.conf_id])

-- Количество выступлений
COUNTUNIQUE([talk_id])

-- Уникальные теги
COUNTUNIQUE([main_tag])

-- Количество выступающих
COUNTUNIQUE([author_id])

-- Средняя длительность (секунды → минуты)
ROUND(AVG(INT([duration])) / 60)

-- Максимум просмотров по конференции
MAX([views_count])

-- Сумма аплодисментов по конференции
SUM([applause_count])

-- Кликабельная ссылка на выступление
URL([url],[title])
```

### Структура данных

**Датасет:** `TS Все таблицы` (объединение таблиц `talks`, `speakers`, `events`)

**Связи таблиц:**
- `talks` ←→ `speakers` по `speaker_id` = `author_id`
- `talks` ←→ `events` по `event_id` = `conf_id`

**Тип соединения:** `INNER JOIN` (только конференции, имеющие хотя бы одно выступление)

## 🔍 Как пользоваться дашбордом

1. **При необходимости используйте фильтры** в верхней части – по стране, конференции, тегу или дате.
2. **Изучите ключевые метрики** – они дают общее представление о масштабе данных.
3. **Исследуйте топ-чарты** – они показывают самые популярные темы и самые смешные выступления.
4. **Детализируйте через таблицы** – кликайте по ссылкам в столбце «название выступления», чтобы посмотреть видео.
5. **Находите инсайты** – например, какое выступление набрало больше всего просмотров или в какой стране самые смешливые зрители.
